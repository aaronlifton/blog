Ever encounter an error like "eslint: '\_myVar' is assigned a value but never
used"? Ideally the `@typescript-eslint/no-unused-vars` rule can be left enabled
and specific variables can be ignored when prefixed with an underscore `_`. You
can do exactly this in a few simple steps.

First, update your `.eslintrc.json` as follows:

\<Code lang="json"
code={`"eslintConfig": {   "rules": {     "no-unused-vars": "off",     "@typescript-eslint/no-unused-vars": [       "warn",       {         "argsIgnorePattern": "^_",         "varsIgnorePattern": "^_",         "caughtErrorsIgnorePattern": "^_"       }   } }`}
marginBottom={true} />

This allows you to prefix any variable with `_` to be ignored by `ESLint`,
including those in destructuring assignments.

So, to review, I was previously getting the following error:

  variable with an underscore to have it be ignored by `ESLint`:

```js annotate
const [_isZoomVisible, setZoomVisible] = useState(false);
//     ^ 
//     [added underscore to variable name] 
```


```js annotate

const blog = "https://jser.dev"
//                    ^
//       [JSer.dev is the **homepage** for JSer.]
//       [Check it out! [jser.dev](https://jser.dev)]

const blog = "https://jser.dev"
//                    ~~~~~~~~
//       [JSer.dev is the homepage for JSer.]
//       [Check it out!]

After updating the `eslintConfig`, the error is now gone and ESLint doesn't
complain about the unused variables:
